How it works

Watch, propose, approve, execute.

A four-step loop that keeps you in control without slowing your agents down.

1
Watch

Watchers poll RSS feeds, Reddit threads, and URL changes on a schedule. Keyword scoring strips noise so only real signals reach your agent. Built-in dedup means no more seen.json files.

2
Propose

The agent drafts an action and pushes it to Impri via MCP tool or a single curl call. Impri stores it, notifies you on your chosen channel, and holds the queue.

3
Approve

You review the action card in your inbox, optionally edit the draft, then approve or reject with one tap from web or mobile. The agent waits — it cannot proceed until you decide.

4
Execute

Only after your approval does the agent run the action — with the exact content you reviewed and possibly edited. It reports the outcome back to close the audit trail.

Why Impri

Built for agents, trusted by humans.

Focused on one thing: holding the queue until you say yes.

Human-in-the-loop safety

Impri never executes anything itself — it holds the queue and signatures; your agent executes after your explicit yes. The gate is in the data, not a prompt. No injection bypasses it: the agent only proceeds after seeing status: "approved" from the API.

Agent-first API

MCP server for Claude Code and any MCP-compatible client. Clean REST API with curl examples, OpenAPI spec, and llms.txt. Idempotent, versioned, no breaking changes in v1. Supports both webhook delivery and polling — agents without a public URL work too.

Open-core, self-hostable

Full inbox + watchers + MCP server under MIT. One docker compose up gives you a running instance with SQLite in under five minutes. No licence required to self-host. Cloud is optional convenience, not a requirement.

Fair pricing

Free tier with 100 approvals per month, no card required. Indie at $9/month. HMAC webhook signing and the base audit log are included on every tier — security features are never paywalled. No charge per agent, no data held hostage.

Quick integration

One call to propose. One decision to execute.

Push the action, poll for the human's decision, execute only on approval. Three curl calls, or three MCP tool calls if you prefer.

REST · https://api.impri.dev/v1 
# 1 — Push an action for approval
curl -X POST https://api.impri.dev/v1/actions \
  -H "Authorization: Bearer $IMPRI_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "kind": "reddit.comment",
    "title": "Reply: Why is resume advice so conflicting?",
    "preview": {
      "format": "markdown",
      "body": "The advice conflicts because different advisors optimise for different audiences..."
    },
    "expires_in": 86400,
    "editable": ["preview.body"]
  }'
# → 201  { "id": "act_abc123", "status": "pending", "inbox_url": "..." }

# 2 — Poll until the human decides (approve/reject from web or mobile)
curl https://api.impri.dev/v1/actions/act_abc123 \
  -H "Authorization: Bearer $IMPRI_API_KEY"
# → { "status": "approved", "decision": { "final_preview": { ... } } }

# 3 — Execute with final_preview (may contain human edits), then close the loop
curl -X POST https://api.impri.dev/v1/actions/act_abc123/result \
  -H "Authorization: Bearer $IMPRI_API_KEY" \
  -d '{"status": "executed"}'
