The loop

Sense the world. Propose. Get a human yes. Then act.

Impri never executes anything itself. It holds the queue and the signatures — your agent does the work, once a person approves.

01

Watch

Watchers poll RSS, Reddit and page changes, score the signal and drop the noise.

02

Propose

Your agent POSTs an action with a preview — a draft reply, an email, a publish.

03

Approve

A human approves, rejects or edits the draft — from the web inbox or their phone.

04

Execute

The decision returns via webhook or polling. Your agent executes — never before.

Why Impri

Built as a safety layer, not a bot.

Human-in-the-loop by design

Fail-closed: on any doubt, don't execute. Edit-before-approve with a signed diff. A decision is immutable and audited.

Agent-first API

MCP tools and a curl-friendly REST API. Idempotent, versioned, with clear failure states. Quickstart in five lines.

Open-core, self-hostable

Run the full core for free with docker compose up and SQLite. No license, no data hostage, full export.

Fair pricing

A free tier you can actually use, no card. Paid tiers cheaper than the hour they save you each month. Security features on every tier.

Five lines

Push an action.
Wait for the human.

One POST puts a draft in the inbox. Your agent polls (or gets a webhook) and only acts on the decision — approved, rejected, or edited. Same flow from an MCP tool.
